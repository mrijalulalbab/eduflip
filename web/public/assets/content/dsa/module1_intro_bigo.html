Apa itu Algoritma?

Algoritma adalah serangkaian langkah-langkah terstruktur untuk menyelesaikan suatu masalah. Bayangkan resep masakan: ada urutan langkah yang harus diikuti untuk menghasilkan hidangan yang diinginkan.

ğŸ’¡ Analogi Sederhana: Jika Anda ingin membuat mie instan, algoritma-nya adalah: (1) Didihkan air, (2) Masukkan mie, (3) Tunggu 3 menit, (4) Tiriskan, (5) Campur bumbu. Setiap langkah harus dilakukan dalam urutan yang benar.

Mengapa Kita Perlu Menganalisis Algoritma?

Tidak semua algoritma diciptakan sama. Ada yang cepat, ada yang lambat. Ada yang hemat memori, ada yang boros. Sebagai programmer, kita perlu tahu seberapa efisien algoritma kita.

Contoh: Mencari nama di buku telepon.

  • Cara Bodoh: Buka halaman 1, cek satu per satu sampai ketemu. (Linear Search)
  • Cara Pintar: Buka di tengah, lihat hurufnya, lalu pilih setengah yang relevan. Ulangi. (Binary Search)

Notasi Big-O: Bahasa Universal Efisiensi

Big-O adalah cara kita mengukur kompleksitas waktu sebuah algoritma. Ia menunjukkan bagaimana waktu eksekusi bertambah seiring bertambahnya input.

Contoh Kode: Menghitung Big-O

1. O(1) - Constant Time

function getFirst(arr) {
    return arr[0]; // Selalu 1 operasi, tidak peduli ukuran array
}

2. O(n) - Linear Time

function printAll(arr) {
    for (let i = 0; i < arr.length; i++) {
        console.log(arr[i]); // Operasi bertambah seiring n
    }
}

3. O(nÂ²) - Quadratic Time

function printPairs(arr) {
    for (let i = 0; i < arr.length; i++) {
        for (let j = 0; j < arr.length; j++) {
            console.log(arr[i], arr[j]); // Loop dalam loop = n * n
        }
    }
}

  • Big-O mengukur efisiensi waktu algoritma.
